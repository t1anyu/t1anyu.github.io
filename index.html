第七章 访问网络文件

主要目的，资源共享。

公用文件夹和共享文件夹。

7.1 公用文件夹

每个登录本地的用户，都可以访问这个文件夹内的内容，这样，就可以不用针对用户设置特定权限了。

默认位于C:\users\public。

解决了本地用户之间资源共享的问题。

可以设置，让网络用户来访问公用文件夹。

\\ip地址

\\192.168.236.138

\\192.168.236.141

本地不同权限的用户，使用这种方式，只能看到自己有权限看到的内容。

默认Everyone完全控制。

更改网络共享设置。

所有网络，启用公用文件夹共享。（加入域后，密码保护的共享不会显示出来，因为这是默认值了。）

专用，公用，域三种类型的网络。

本地用户只能访问本地公用文件夹，如果访问网络上的公用文件夹，需要使用网络用户身份。

7.2 共享文件夹

可将通过将文件夹共享的方式，让网络用户访问特定文件夹内的文件和文件夹。

7.2.1 共享文件夹的权限

读取

更改

完全控制

只有通过网络形式来访问，此权限才起作用。

本地登录，不受限制。（注，输入\\ip地址，视为网络形式。）

7.2.2 用户的有效权限

权限累加

“拒绝”权限优先级较高

共享文件夹的移动和复制，复制的文件，不再共享；移动到其他磁盘分区，不再共享。

与NTFS权限搭配使用

注意，这里有两个权限在一起起作用，不要搞混了。

共享权限解决能否通过网络访问，以及通过网络访问后权限的大小；

本地权限会最终影响到对文件的访问权限。

网络用户的最终权限，将是网络权限和本地权限两者最严格的的权限设置。

如果没有通过网络登录，那么，共享权限不起作用，只有本地（NTFS）权限起作用。

7.4 访问网络公用和共享文件夹的方式

7.4.1 利用网络发现

前提：必须启用网络发现功能。

未启用：点网络图标，单击上方的提示，启用网络发现。

附注：

网络和共享中心，更改高级共享设置。

Win7客户端，开始->计算机->网络

开始->系统管理工具->服务

Function Discovery Resource Publication服务是否启用

总结：

1.要启用网络发现。

2.启用Function Discovery Resource Publication服务。

连接计算机的网络验证机制

操纵的计算机->网络计算机

发送在操纵的计算机上登录的用户名和密码给网络计算机。

1.加入域

将密码发送给网络计算机后，网络计算机将登录信息发送给DC，DC验证通过后，可登录。

2.未加入域（不受信任的网络中）

如果网络计算机上，有和在操纵计算机上一致的用户名和密码，则，使用网络计算机上的这个同名的账户来登录。

如果有同名的，但密码不同，则要求输入正确的登录信息。

如果网络计算机上没有和在操纵计算机上一致的用户名和密码，且启用了guest用户，则登录者的身份为guest。

管理网络密码

控制面板->用户账户->凭据管理器->Windows凭据->可修改，添加、删除凭据(如果是域，用户名前要加入域名)

如果在某次登录时，勾选了记住我的凭据，此凭据也会中凭据管理器中被看到。

7.5 脱机文件

让计算机在未与公司网络连接情况下，访问原本位于网络计算机中的文件（网络文件）。

访问的是网络文件的缓存(副本)版本。

缓存文件和网络文件需要同步：

缓存文件更改过，网络文件未更改，缓存文件->网络文件。

缓存文件未更改过，网络文件更改，网络文件->缓存文件。

缓存文件和网络文件都更改过，系统会让你选择保留一个，或将两个都保留。

7.5.1 网络计算机端的脱机文件设置

前提：存放文件的文件夹，必须是共享文件夹。

步骤：

共享文件夹上右击->属性->共享->高级共享->缓存

7.5.2 网络用户如何使用脱机文件

1.启用脱机文件

Windows7,Windows8默认已经启用了脱机文件。

Windows Server 2008 R2、Windows Server 2012 R2需要先通过服务器管理器来添加桌面体验功能。

到了选择角色界面，单击下一步(我们需要安装得是一个功能，而不是一个角色)，选择用户界面和基础结构，勾选里面的桌面体验。

重启后，按照如下步骤启用脱机文件。

控制面板->将查看方式修改为大图标或小图标->同步中心->管理脱机文件->启用脱机文件->重启电脑。

2.选择可脱机文件

3.同步处理

4.测试脱机文件是否正常工作

通过禁用网卡的方式，来模拟断网的情况。（需要管理员权限）
